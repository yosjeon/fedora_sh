```
bcc() {
```

```
$ echo "$(bcc "958742216 + 365453312") #-- c01"
1,324,195,528 #-- c01
```

## 🔥 bash 터미널에서 간단한 연산을 하고 싶어서 bc 를 쓰는데,
숫자값에 콤마 표시가 안되네. bc의 결과에 콤마를 넣는 옵션이 따로 있나?

### 🧩 아쉽게도, 표준 bc 명령어 자체에는 계산 결과에 **천 단위 구분 기호(콤마)**를
자동으로 추가하는 직접적인 옵션은 없습니다. bc는 주로 정밀한 수학 연산을 위해 설계되었으며, 출력 형식 지정 기능은 제한적입니다. 🔢
### 💡 해결 방법: 파이프라인으로 printf 명령어 사용
bash 셸 환경에서 bc의 결과에 콤마를 넣으려면, bc의 출력을 다른 셸 명령어인 **printf**나 numfmt 등의 포맷팅 도구로 **파이프(|)**를 통해 전달하여 처리해야 합니다.

가장 일반적인 방법은 printf 명령어를 로케일 설정과 함께 사용하는 것입니다.
#### 1. printf와 로케일(Locale) 사용
printf 명령어에서 ' 플래그는 활성화된 로케일에 따라 천 단위 구분 기호를 사용하도록 지시합니다. 한국(ko_KR) 로케일은 보통 콤마를 쓰지 않으므로, 콤마를 쓰는 로케일(예: 미국 en_US.UTF-8)을 일시적으로 지정해 주어야 합니다.
```Bash
# 예시: 12345678.9012 계산 결과에 콤마 추가
$ echo "scale=4; 12345678.9012 + 0" | bc | xargs -I {} printf "LC_NUMERIC=en_US.UTF-8 %'.f\n" {}

# 출력 결과:
# 12,345,679
```
#### 주의 사항:
- %'.f 포맷은 숫자를 정수로 간주하고 로케일에 따라 콤마를 추가한 후 출력합니다. 따라서 소수점 이하는 반올림되어 사라집니다. (위 예시에서는 소수점 첫째 자리에서 반올림되어 12345679가 됩니다.)
- 소수점 이하 자릿수까지 유지하려면, bc에서 얻은 소수점 아래 자릿수를 직접 계산해서 %.Nf 포맷에 넣어줘야 합니다. 이 과정은 매우 복잡해지므로, bc를 사용하는 목적(단순 연산)을 넘어섭니다.
#### 2. 소수점도 살리고 싶다면
bc의 출력을 받아 **sed**나 awk 같은 텍스트 처리 도구를 사용하여 직접 정규 표현식으로 콤마를 삽입하는 것이 더 유연할 수 있습니다.
```Bash
# 예시: bc 결과에 콤마와 소수점 이하까지 유지 (sed 사용)
$ echo "12345678.9012 * 1" | bc | sed ':a;s/\B[0-9]\{3\}\(\.[0-9]*\)\?\%*$/,&/g;ta'

# 출력 결과:
# 12,345,678.9012
```
이 sed 명령어는 복잡하지만, 소수점 앞의 정수 부분에만 천 단위 콤마를 추가하고 소수점 이하 부분은 그대로 유지합니다.

단순히 계산만 필요하다면 bc를 그대로 사용하시고, 콤마 표시가 꼭 필요하다면 위와 같이 다른 명령어를 파이프로 연결하여 후처리하는 방식을 이용하셔야 합니다.

혹시 bc에서 계산한 특정 결과에 대해서만 콤마를 적용하는 방법을 더 자세히 알고 싶으신가요?

그러면, 이걸 .bashrc 에 alias 로 넣어두려면 어떻게 하지?

### 🧩 bc의 결과에 콤마를 붙여주는 기능을 .bashrc에 등록하여 편리하게 사용할 수 있습니다.
복잡한 파이프라인 처리를 위해 간단한 셸 함수(Shell Function) 형태로 등록하는 것을 추천합니다.
### 1. .bashrc에 함수 등록하기
텍스트 편집기(예: nano, vi, vim)를 사용하여 ~/.bashrc 파일을 열고, 파일의 맨 아래에 다음 코드를 추가하세요. 함수 이름은 bcc (bc with comma) 등으로 지정할 수 있습니다.
```Bash
# bc 계산 결과에 천 단위 구분 기호(콤마)를 추가하는 함수
# LC_NUMERIC=en_US.UTF-8을 사용하여 콤마를 천 단위 구분 기호로 설정합니다.
# %'.0f 포맷을 사용하여 정수형으로 표시하고 콤마를 추가합니다.
bcc() {
  echo "$@" | bc | xargs -I {} printf "LC_NUMERIC=en_US.UTF-8 %'.0f\n" {}
}
```
#### 코드 설명
| 부분 | 역할 |
|:---:|:---:|
| bcc() { ... } | bcc라는 이름의 셸 함수를 정의합니다. |
| echo "$@" | 함수에 전달된 모든 인자(예: "12345678 * 2")를 bc로 전달합니다. |
| `bc` | | 
| `xargs -I {}` | |
| LC_NUMERIC=en_US.UTF-8 | printf가 숫자를 포맷팅할 때 콤마(,)를 천 단위 구분 기호로 사용하도록 로케일 환경을 일시적으로 설정합니다. |
| %' | 로케일 설정에 따른 천 단위 구분 기호를 활성화합니다. |
| .0f\n | 결과를 소수점 이하 0자리의 실수(f)로 출력(즉, 정수형)하고 줄 바꿈(\n)을 추가합니다. |
### 2. 변경 사항 적용 및 사용
.bashrc 파일에 내용을 추가했다면, 변경된 내용을 현재 터미널 세션에 적용해야 합니다.
```Bash
$ source ~/.bashrc
```
이제 터미널에서 bcc 함수를 사용하여 연산을 수행해 보세요.
#### 사용 예시
```Bash
$ bcc "12345678 * 10"

$ bcc "987654321 + 123456789"

```
함수를 이용하면 bc의 강력한 연산 기능과 콤마 구분 기호가 적용된 깔끔한 출력 결과를 동시에 얻을 수 있습니다.

### 🧩 네, 값을 전달할 때 따옴표를 사용하는 것이 좋습니다. 👍

함수를 다음과 같이 정의했습니다:
```Bash
bcc() {
  echo "$@" | bc | xargs -I {} printf "LC_NUMERIC=en_US.UTF-8 %'.0f\n" {}
}
```
### 🧐 따옴표를 사용해야 하는 이유
bc에 전달하는 수식에는 보통 *, +, -, /, % 등의 연산자가 포함됩니다. 이 연산자들은 bash 셸에서 특별한 의미를 가집니다.
#### 1. 와일드카드 확장 방지 (*)
*는 셸에서 현재 디렉토리의 모든 파일 이름과 일치하는 **와일드카드(glob)**로 해석됩니다.
- 따옴표 사용 (권장):
```Bash
$ bcc "1234 * 10"  # "1234 * 10" 전체가 bc로 정확히 전달됨
12,340
```
- 따옴표 미사용 (문제 발생):
```Bash
$ bcc 1234 * 10
# 셸이 *를 파일 목록으로 확장하려 시도. 
# 예: 만약 'file1', 'file2' 파일이 있다면, 'bcc 1234 file1 file2 10'으로 실행되어 bc 오류 발생.
```
#### 2. 공백을 하나의 인수로 묶기
따옴표("...")를 사용하면 수식 전체(12345678 * 10)가 bcc 함수의 단일 인수로 묶여서 echo "$@"에 의해 bc로 안전하게 전달됩니다. 따옴표를 사용하지 않으면 각 숫자와 연산자가 별개의 인수로 분리될 수 있습니다.
#### 3. 변수 확장 보존
수식 내부에 셸 변수를 사용하더라도, 이중 따옴표(")를 사용하면 변수 값은 확장되면서도 전체 수식의 구조는 보존됩니다.

따라서, bc 연산을 위한 수식은 항상 이중 따옴표(")로 묶어서 bcc 함수에 전달하는 습관을 들이시는 것이 가장 안전하고 확실한 방법입니다.

### 🧩 요청하신 기능을 vi/vim에서 구현하려면
**Vim 스크립트(Vimscript)**를 사용하여 **매핑(mapping)**과 **함수(function)**를 정의해야 합니다. Ctrl + ??? 형태의 단축키를 이용해 텍스트를 입력하고, 해당 텍스트를 기반으로 일련번호가 매겨진 Q/A 쌍을 자동으로 생성하는 함수를 만들 수 있습니다.다만, Vim은 기본적으로 마우스로 긁어서 텍스트를 기억하는 기능을 매핑에서 직접 활용하기 어렵습니다. 대신, Vim의 레지스터(Register) 또는 명령어 라인 입력을 활용하는 방법을 사용해야 합니다.가장 유연하고 강력한 방법은 **명령어 모드(:)**에서 인수를 입력하고 실행하는 함수를 정의하는 것입니다.
### Vim/Vimscript를 이용한 자동 일련번호 생성 (추천 방법)
이 방법은 명령어 라인에서 :{과제명} {시작번호} '{소제목}' [개수] 형태로 입력하고 실행합니다.
#### 1. Vimrc 설정 (함수 정의)
다음 내용을 $HOME/.vimrc 파일에 추가하거나, 별도의 .vim 파일로 저장 후 source 명령으로 로드합니다.
```코드 스니펫
" Q/A 쌍을 지정된 형식으로 생성하는 함수
function! GenerateQAPairs(job_name, start_num, subtitle, count)
    " 현재 시간을 포맷에 맞게 가져오기 (쉘 명령 실행)
    " %m%d_%a.%H%M 형식으로 날짜/시간 포맷
    let l:date_time = system("LC_TIME=C date +%m%d_%a.%H%M")
    " system() 결과에는 개행문자가 포함되어 있으므로 제거
    let l:date_time = substitute(l:date_time, '\n', '', '')

    " 생성할 텍스트를 저장할 리스트 초기화
    let l:output = []

    " 지정된 개수만큼 반복
    for i in range(0, a:count - 1)
        let l:current_num = a:start_num + i
        " 분류번호를 최소 2자리로 포맷 (예: 00, 01, ..., 99, 100)
        let l:seq_num = printf("%02d", l:current_num)

        " 최종 포맷 문자열 생성
        let l:base_format = printf(".%s-%s.%s", l:date_time, l:seq_num, a:subtitle)
        
        " Q와 A 쌍 추가
        call add(l:output, "Q." . a:job_name . l:base_format)
        call add(l:output, "A." . a:job_name . l:base_format)
        call add(l:output, "") " 빈 줄 추가
        call add(l:output, "") " 빈 줄 추가 (다음 Q/A 쌍과의 간격)
    endfor

    " 마지막에 추가된 빈 줄 제거 (선택 사항)
    let l:output = l:output[:-2]

    " 현재 커서 위치에 생성된 텍스트 삽입
    call append(line('.'), l:output)

    " (선택 사항) 삽입 후 커서를 첫 Q의 위치로 이동
    execute "normal! " . line('.') . "G"
endfunction

function! s:ParseAndGenerate(...)
    let args = a:000
    if len(args) < 3
        echohl ErrorMsg | echo "사용법: :GenQA <과제명> <시작번호> '<소제목>' [개수]" | echohl None
        return
    endif
    
    let job_name = args[0]
    let start_num = str2nr(args[1])
    let subtitle = substitute(args[2], ['^''', ''']$'], ['', ''], 'g') " 따옴표 제거
    let count = (len(args) > 3) ? str2nr(args[3]) : 10 " 기본 10개

    call GenerateQAPairs(job_name, start_num, subtitle, count)
endfunction
```

### 쉘 스크립트 (mytitle.sh)를 이용한 자동 생성 (추천)
#### 1. 스크립트 파일 생성
다음 내용을 mytitle.sh 파일로 저장합니다.
```Bash
#!/bin/bash

# 사용법: mytitle <과제명> <시작번호> "<소제목>" [마지막번호]

# 최소 3개의 인수가 필요합니다.
if [ "$#" -lt 3 ]; then
    echo "사용법: mytitle <과제명> <시작번호> \"<소제목>\" [마지막번호]" >&2
    echo "예시: mytitle jobeee 00 \"good day morning\" 12" >&2
    exit 1
fi

JOB_NAME="$1"
START_NUM=$(printf "%d" "$2") # 시작 번호를 정수로 변환
SUBTITLE="$3"

# 소제목의 공백을 밑줄(_)로 변환합니다.
CLEAN_SUBTITLE=$(echo "$SUBTITLE" | tr ' ' '_')

# 마지막 번호를 설정합니다. (입력되지 않으면 시작번호와 동일)
if [ -z "$4" ]; then
    END_NUM=$START_NUM
else
    END_NUM=$(printf "%d" "$4")
fi

# 현재 시각을 포맷에 맞게 설정합니다.
# LC_TIME=C를 사용하여 요일을 영문으로 고정합니다.
DATE_TIME=$(LC_TIME=C date +%m%d_%a.%H%M)

# 시작 번호부터 마지막 번호까지 반복하며 Q/A 쌍을 생성합니다.
for (( i=$START_NUM; i<=$END_NUM; i++ )); do
    # 일련번호를 최소 2자리로 포맷합니다. (예: 00, 01, ..., 99, 100)
    SEQ_NUM=$(printf "%02d" $i)
    
    # 최종 문자열 포맷
    BASE_STR="${JOB_NAME}.${DATE_TIME}-${SEQ_NUM}.${CLEAN_SUBTITLE}"
    
    # Q/A 쌍 출력
    echo "Q.${BASE_STR}"
    echo "A.${BASE_STR}"
    echo ""
    echo ""
done
```

### 🔥 잘 되네. 그리고, 위키에서 $\to$ 를 입력하면 화살표가 나타나네. 처음 봤어.

### 🧩 놀라운 경험을 하셨네요! 말씀하신 $\to$와 같은 화살표 기호는 LaTeX(라텍) 문법으로 입력된 것입니다.
LaTeX는 주로 수학, 과학 문서나 전문적인 문서 작성을 위해 사용되는 조판 시스템이며, 특히 수학 기호, 특수 문자, 복잡한 수식을 표현하는 데 매우 강력합니다. 많은 위키나 Markdown 기반 환경(예: Jupyter Notebook, 일부 온라인 편집기)에서 수식을 표현하기 위해 LaTeX 문법을 지원하고 있습니다.
이러한 특수 기호와 수식 입력 방법을 볼 수 있는 곳은 크게 세 가지입니다.
### 1. LaTeX 수학 기호 목록
$\to$ 와 같은 기호는 LaTeX에서 수학 기호를 나타내는 명령어로, 이들을 모아 놓은 목록을 참고하시면 됩니다.
- 웹 검색: "LaTeX 기호 목록", "LaTeX symbol list", "LaTeX math symbols" 등의 키워드로 검색하시면 방대한 목록을 찾을 수 있습니다.
- 주요 기호 예시:
  - $\to$ (화살표): \to 또는 \rightarrow
  - $\leftarrow$ (왼쪽 화살표): \leftarrow
  - $\leftrightarrow$ (양방향 화살표): \leftrightarrow
  - $\sum$ (시그마): \sum
  - $\sqrt{x}$ (제곱근): \sqrt{x}
  - $\alpha, \beta, \gamma$ (그리스 문자): \alpha, \beta, \gamma
### 2. 위키/Markdown 수식 문서
  특정 위키(예: 나무위키, 위키피디아)나 Markdown 환경에서 어떤 LaTeX 명령어를 지원하는지 안내하는 문서를 참고할 수 있습니다.
- 나무위키: "나무위키 수식" 또는 "나무위키 특수 문자" 등으로 검색하시면 해당 위키에서 사용하는 수식 문법(주로 MathJax 기반의 LaTeX)에 대한 상세한 가이드를 볼 수 있습니다.
- GitHub/Jupyter Notebook: 이들 환경에서 사용되는 Markdown의 수식 입력 방식을 안내하는 문서를 찾아보면 됩니다. 보통 $나 $$로 감싸서 LaTeX 명령어를 입력합니다.
### 3. 유니코드 및 한자 키 입력 (보편적인 특수 문자)
LaTeX와 별도로, 자주 사용되는 일반적인 특수 문자는 운영체제나 한글 입력기의 기능을 통해 입력할 수 있습니다.
- 한글 자음 + 한자 키 변환: 한국어 입력 환경에서 가장 흔히 사용하는 방법입니다.
  - ㅁ + 한자 키: 일반 기호 (★, ☎, ※, $\to$ $\times$ $\to$ 이 화살표는 유니코드 화살표가 아닐 수 있습니다)
  - ㄱ + 한자 키: 문장 부호 ('', 『』, ·, …)
  - ㄷ + 한자 키: 숫자/수학 기호 ($+$, $\times$, $\div$, $\le$)
  - ㄹ + 한자 키: 단위/통화 기호 ($, $, ₩, ℃)
이 중에서 1번 LaTeX 기호 목록을 검색해 보시면, $\to$ 와 같은 멋진 기호들을 가장 많이 접하실 수 있을 겁니다.


## 🔥 PuTTY 0.83 을 설치하다가 잘못 삭제해서 다시 설치하려고 하니
설치를 실행하다가 "Another installation is in progress. You must complete that installation before continuing this one." 이라고 뜨네.
그런데 지금 윈도우 창에는 떠있는 프로세스가 없거든.
그 숨은 프로세스를 찾아서 삭제하려면 어떻게 하면 되지?

### 🧩💡 해당 오류는 Windows Installer 서비스가 이전 설치 작업이 아직 진행 중이라고 판단했을 때 발생합니다.
프로세스 목록에 직접 보이지 않아도 내부적으로 설치 상태가 남아있을 수 있어요.

가장 빠르고 확실한 해결 방법은 컴퓨터를 재부팅하는 것이지만, 이미 시도했거나 재부팅 없이 해결하고 싶으신 경우 아래 단계를 따라 숨겨진 설치 프로세스를 종료하고 상태를 초기화할 수 있습니다.

### 🛠️ 설치 진행 중 오류 해결 단계
#### 1. msiexec.exe 프로세스 강제 종료
Windows Installer 프로세스인 msiexec.exe가 백그라운드에서 실행 중일 가능성이 높습니다.
- Ctrl + Shift + Esc를 눌러 작업 관리자를 엽니다.
- 창 하단의 **'자세히'**를 클릭합니다 (이미 자세히 보기라면 생략).
- '세부 정보' 탭으로 이동합니다.
- 목록에서 **msiexec.exe**를 찾습니다.
  - 참고: 이름이 'Windows Installer'일 수도 있습니다.
- 해당 프로세스를 선택하고 오른쪽 아래의 '작업 끝내기' 버튼을 누릅니다.
- msiexec.exe가 모두 사라졌는지 확인합니다.
#### 2. Windows Installer 서비스 재시작
설치 서비스 자체를 초기화하는 방법입니다.
- Win + R을 누르고 **services.msc**를 입력 후 Enter를 칩니다.
- 서비스 창에서 목록을 아래로 스크롤하여 'Windows Installer' 서비스를 찾습니다.
- 해당 서비스를 마우스 오른쪽 버튼으로 클릭하고 **'다시 시작'**을 선택합니다.
  - '다시 시작' 옵션이 없다면, **'중지'**를 선택한 후 다시 마우스 오른쪽 버튼을 클릭하여 **'시작'**을 선택합니다.
#### 3. 레지스트리 항목 확인 및 삭제 (주의 필요)
위의 방법으로 해결되지 않았다면, 레지스트리에 설치 상태 정보가 남아있는 경우일 수 있습니다. 레지스트리 편집은 시스템 안정성에 영향을 줄 수 있으므로 반드시 주의해서 진행해야 하며, 다른 설치가 진행 중이 아닌 것이 확실할 때만 시도하세요.
- Win + R을 누르고 **regedit**를 입력 후 Enter를 칩니다.
- 다음 경로로 이동합니다:
```
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Installer
```
- Installer 키 아래에 **InProgress**라는 키가 있는지 확인합니다.
- 만약 InProgress 키가 있다면, 해당 키를 선택하고 삭제합니다.
- 다음 경로도 확인합니다 (경로가 없을 수도 있습니다):
```
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Installer
```
- Installer 키 아래에 **InProgress**라는 키가 있다면 동일하게 삭제합니다.
- 레지스트리 편집기를 닫고 PuTTY 설치를 다시 시도합니다.

대부분의 경우 1단계(msiexec.exe 종료) 또는 재부팅만으로 문제가 해결됩니다. 안전을 위해 먼저 1단계와 2단계를 시도해 보시는 것을 권장합니다.

``` 
import tkinter as tk
from tkinter import simpledialog, messagebox

class PomodoroApp:
    def __init__(self, root):
        self.root = root
        self.root.withdraw()  # Start with the main window hidden
        self.timer_job = None
        self.show_main_dialog()

    def show_main_dialog(self):
        """Shows the initial dialog to get the timer duration."""
        # Center the root window first, so the simpledialog (which is parented to it) also appears centered.
        # The simpledialog's actual size is determined by its content, but centering the parent helps.
        self.center_window(self.root, 300, 150)
        self.root.deiconify() # Temporarily show the root window
        self.root.attributes('-topmost', True) # Make root topmost
        self.root.lift() # Bring it to the front

        minutes = simpledialog.askinteger("Pomodoro Timer",
                                          "Enter duration in minutes (default: 25):",
                                          parent=self.root,
                                          minvalue=1,
                                          initialvalue=25)

        self.root.attributes('-topmost', False) # Revert topmost
        self.root.withdraw() # Hide the root window again

        if minutes is not None:
            self.start_timer(minutes)
        else:
            self.root.quit()

    def start_timer(self, minutes):
        """Starts the countdown timer."""
        self.root.title(f"Pomodoro: {minutes} min")
        self.root.deiconify()  # Show the window in the taskbar
        self.root.withdraw()   # But keep it hidden from view

        seconds = minutes * 60
        self.countdown(seconds)

    def countdown(self, seconds_left):
        """The main countdown loop, running every second."""
        if seconds_left > 0:
            # Update title to show progress (optional, but nice to have)
            mins, secs = divmod(seconds_left, 60)
            self.root.title(f"Pomodoro - {mins:02d}:{secs:02d}")

            # Schedule the next update
            self.timer_job = self.root.after(1000, self.countdown, seconds_left - 1)
        else:
            self.root.title("Pomodoro - Time's up!")
            self.show_alert()

    def show_alert(self):
        """Shows the 'always on top' alert window when the timer is done."""
        alert_window = tk.Toplevel(self.root)
        alert_window.title("Time's Up!")

        # Make the window always on top
        alert_window.attributes("-topmost", True)

        # Center the window
        self.center_window(alert_window, 300, 150)

        message = "Time is up! What would you like to do?"
        tk.Label(alert_window, text=message, padx=20, pady=20).pack()

        button_frame = tk.Frame(alert_window)
        button_frame.pack(pady=10)

        def restart():
            alert_window.destroy()
            if self.timer_job:
                self.root.after_cancel(self.timer_job)
            self.show_main_dialog()

        def close_app():
            alert_window.destroy()
            self.root.quit()

        tk.Button(button_frame, text="Restart", command=restart).pack(side="left", padx=10)
        tk.Button(button_frame, text="Close", command=close_app).pack(side="right", padx=10)

        # When the alert window is closed via 'X', treat it as closing the app
        alert_window.protocol("WM_DELETE_WINDOW", close_app)
        alert_window.focus_force()

    def center_window(self, win, width, height):
        """Helper function to center a window on the screen."""
        screen_width = win.winfo_screenwidth()
        screen_height = win.winfo_screenheight()
        x = (screen_width/2) - (width/2)
        y = (screen_height/2) - (height/2)
        win.geometry('%dx%d+%d+%d' % (width, height, x, y))


if __name__ == "__main__":
    main_root = tk.Tk()
    app = PomodoroApp(main_root)
    main_root.mainloop()
```

## 🔥 windows 에서 Git-Bash 를 쓰고 있는데, 여기서 rsync 를 쓰려면 어떻게 하나?

🔋
### 🔋 Git Bash는 기본적으로 rsync를 포함하고 있지 않기 때문에, 사용하려면 수동으로 설치해야 합니다.
Git for Windows는 MSYS2를 기반으로 하므로, MSYS2 저장소에서 rsync 바이너리를 다운로드하여 Git Bash 환경에 통합하는 방법이 일반적입니다.

다음은 일반적인 설치 단계입니다. 이 과정은 시스템 파일(Git 설치 폴더)에 접근하고 파일을 복사해야 하므로 관리자 권한이 필요할 수 있습니다.

### 1. rsync 및 종속성 파일 다운로드
rsync는 보통 MSYS2 저장소에서 다운로드합니다. 이 저장소에는 .pkg.tar.zst 형식의 패키지가 있습니다.
#### 1. rsync 패키지 다운로드:
  - MSYS2 저장소 (http://repo.msys2.org/msys/x86_64/) 에서 가장 최신 버전의 `rsync-*.pkg.tar.zst` 파일을 찾아서 다운로드합니다.
#### 2. 종속성 패키지 다운로드:
  - rsync가 실행되기 위해 필요한 libxxhash 및 libzstd 와 같은 종속성 파일도 다운로드해야 할 수 있습니다. MSYS2 저장소에서 libxxhash-*.pkg.tar.zst 와 libzstd-*.pkg.tar.zst 파일을 찾아서 다운로드합니다.
### 2. 패키지 압축 해제
다운로드한 파일들은 Zstandard(zstd) 압축 방식으로 압축되어 있습니다.
#### 1. 압축 해제 도구 준비:
  - 7-Zip과 같은 범용 압축 해제 도구를 사용하여 .zst 파일을 해제할 수 있습니다. 7-Zip이 .zst 파일을 직접 해제하지 못하는 경우, 먼저 zstd 실행 파일을 다운로드하여 Git Bash 환경에 통합해야 합니다.
#### 2. 압축 해제:
  - 각 .pkg.tar.zst 파일을 해제하면 보통 내부에 `*.tar` 파일이 나오고, 이 `*.tar` 파일을 다시 해제하면 usr 폴더 구조가 나타납니다.
  - 최종적으로 필요한 파일은 rsync.exe (주로 usr/bin 폴더 안에 있음)와 .dll 파일들 (주로 usr/bin 폴더 안에 있음)입니다.
### 3. Git Bash 폴더에 파일 복사
Git Bash는 Git for Windows가 설치된 디렉토리의 usr/bin 폴더에 실행 파일을 저장합니다.
#### 1. 대상 폴더 확인:
  - 일반적으로 Git for Windows 설치 경로는 C:\Program Files\Git 입니다.
  - rsync와 필요한 DLL 파일을 복사할 폴더는 C:\Program Files\Git\usr\bin 입니다.
#### 2. 파일 복사:
  - 압축을 풀어 얻은 rsync.exe 파일을 위 대상 폴더에 복사합니다.
  - 마찬가지로, 압축을 풀어 얻은 모든 필요한 .dll 종속성 파일 (예: msys-zstd-*.dll, msys-xxhash-*.dll 등)을 대상 폴더에 복사합니다.
### 4. 설치 확인
Git Bash를 새로 열고 다음 명령어를 실행하여 rsync가 제대로 설치되었는지 확인합니다.
```Bash
rsync --version
```
버전 정보가 출력되면 성공적으로 설치된 것입니다.
### 💡 대안적인 방법 (WSL 또는 다른 도구)
만약 위 방법이 복잡하게 느껴지거나 호환성 문제가 발생한다면, 다음과 같은 대안을 고려해 볼 수 있습니다.
- `WSL (Windows Subsystem for Linux)`: Windows 10/11 사용자라면 WSL을 설치하고, WSL 환경 내에서 Linux의 **apt**나 `yum` 같은 패키지 관리자를 사용하여 rsync를 설치할 수 있습니다. rsync를 안정적으로 사용할 수 있는 가장 좋은 방법 중 하나입니다.
- `Cygwin`: Cygwin을 설치할 때 rsync 패키지를 선택하여 설치하고, 이를 Git Bash의 PATH에 추가하여 사용할 수도 있습니다. (Git Bash는 MinGW를 기반으로 하지만, Cygwin의 도구를 통합하여 사용하는 경우가 있습니다.)
- `MinGW-w64/MSYS2 단독 사용`: Git Bash 대신 정식 MSYS2 환경을 설치하고 거기서 pacman -S rsync 명령어로 rsync를 설치하여 사용하는 방법도 있습니다.

### 🔋 WSL(Windows Subsystem for Linux)을 설치하는 가장 쉽고 권장되는 방법은 단일 명령어를 사용하는 것입니다.
이 방법은 Windows 10 버전 2004(빌드 19041) 이상 또는 Windows 11에서 사용할 수 있습니다.
### 🚀 WSL 자동 설치 명령어
#### 1. 관리자 권한으로 터미널 실행:
  - Windows 시작 메뉴에서 PowerShell 또는 명령 프롬프트(CMD), 또는 Windows Terminal을 검색한 후, **"관리자 권한으로 실행"**을 선택합니다.
#### 2. 설치 명령어 입력:
  - 다음 명령어를 입력하고 엔터를 누릅니다.
```Bash
wsl --install
```
- 이 명령어는 다음 작업을 자동으로 수행합니다:
  - WSL 및 가상 머신 플랫폼과 같은 필요한 Windows 선택적 기능을 활성화합니다.
  - WSL 커널을 다운로드하고 설치합니다.
  - 기본 Linux 배포판(일반적으로 Ubuntu)을 설치합니다.
#### 3. 컴퓨터 재시작:
  - 설치가 완료되면 컴퓨터를 재시작합니다.
#### 4. Linux 설정:
  - 재시작 후, 설치된 Linux 배포판이 자동으로 실행되어 사용자 이름과 암호를 설정하라는 메시지가 나타납니다. 지시에 따라 설정하면 WSL 설치가 완료됩니다.
### 🛠️ 추가 명령어 및 팁
| 명령어 | 설명 |
|----|----|
| wsl --list --online | 설치 가능한 Linux 배포판 목록을 확인합니다. |
| wsl --install -d <배포판이름> | 특정 배포판을 설치합니다. (예: wsl --install -d Debian) |
| wsl --set-default-version 2 | WSL 설치 시 기본 버전을 WSL 2로 설정합니다. (자동 설치 시 기본값은 2입니다.) |
| wsl -l -v | 설치된 WSL 배포판 목록과 버전(1 또는 2) 및 상태를 확인합니다. |

### 🔋 Dart는 앱을 만드는 데 사용되는 프로그래밍 언어이고,
Flutter는 Dart 언어를 사용하여 여러 플랫폼(iOS, 안드로이드, 웹, 데스크톱 등)의 앱 UI를 만드는 크로스 플랫폼 프레임워크입니다. 즉, Dart는 언어 자체이며 Flutter는 이 언어를 기반으로 만들어진 개발 도구입니다. 
### Dart
- `언어`: Google이 개발한 객체 지향 프로그래밍 언어입니다.
- `특징`:
	- UI 개발에 최적화되어 있으며, 웹, 서버, 모바일 등 다양한 플랫폼에서 사용할 수 있습니다.
- 널 세이프티(null-safety)와 같은 기능으로 더 안정적인 코드를 작성할 수 있습니다.
- 다양한 라이브러리와 풍부한 커뮤니티 지원을 제공합니다. 
### Flutter
- `프레임워크`: Dart 언어를 기반으로 하며, UI 툴킷입니다.
- `특징`:
	- 하나의 코드로 iOS, 안드로이드, 웹, 데스크톱 등 여러 플랫폼에서 실행되는 앱을 만들 수 있습니다.
- 핫 리로드 기능으로 코드 변경 사항을 즉시 확인할 수 있어 개발 속도가 매우 빠릅니다.
- 위젯 기반의 UI를 쉽게 구축할 수 있으며, 자체 렌더링 엔진을 사용해 각 플랫폼에 최적화된 UI를 구현합니다.
- 다양한 플랫폼의 네이티브 기능과 연동하기 위해 Platform Channels를 제공합니다.

- 특징:
  - 간단하고 시각적으로 만족스러운 디자인의 타이머입니다. (Electron 기반)

  - 다양한 테마와 사용자 정의 테마 기능을 제공합니다.
#### 3. Stretchly
- 특징:
  - Pomodoro 뿐만 아니라 규칙적인 휴식을 상기시키는 데 초점을 맞춘 프로그램입니다. (Micro-break, Long break)

## 🔥 고문진보에서 "XX는 고군이요 XX는 YY이라" 라는 구절을 찾으려고 하는데,
XX 는 기억이 안나고, YY 는 '신촌'인지 '신도'인지 명확하지 않아. 이걸 어디서 찾아야 되지?

🔋
### 🔋 찾으시는 구절은 **'등왕각서(滕王閣序)'** 에 나오는 문구일 가능성이 매우 높습니다.

검색 결과, 『고문진보』에 수록된 당나라 왕발(王勃)의 **「등왕각서」** 에 다음과 같은 구절이 있습니다.
```
南昌故郡, 洪都新府.
(남창은 **고군(故郡)**이요, 홍도는 **신부(新府)**라.)
```
이 구절이 변형되어 암기되었거나, 지역에 따라 다르게 인용되는 과정에서 "XX는 고군이요 XX는 YY이라"와 유사한 형태가 된 것으로 보입니다.
### 구절의 유래 및 원문
- `원전`: 『고문진보』 전집(前集)에 수록된 「등왕각서(滕王閣序)」
- `작가`: 왕발(王勃) (당나라 초기)
- `원문`: 南昌故郡, 洪都新府. (남창은 고군(故郡)이요, 홍도는 신부(新府)라.)

따라서, 기억나지 않던 'XX'는 **남창(南昌)** 과 **홍도(洪都)** 이며, 'YY'는 **신부(新府)** 일 가능성이 가장 큽니다.
- '신촌'이나 '신도'가 아닌 **'신부(新府)'** 가 원문입니다.
  - `고군(故郡)`: 오래된 고을 (옛 군현의 이름)
  - `신부(新府)`: 새로 설치된 도호부(都護府)나 부(府)의 이름

이 구절은 등왕각이 있는 **남창(南昌)** 이 예전에는 고군이었지만, 지금은 홍도라는 신부로 불리게 되었다는 내용을 담고 있어, 웅장한 등왕각의 배경을 설명하는 첫 문장으로 사용됩니다.

### 🔋 Chrome 브라우저에서 YouTube 자막을 다운로드하는 가장 일반적이고 간편한 방법은
Chrome 확장 프로그램을 이용하거나 온라인 자막 다운로더 웹사이트를 사용하는 것입니다.

yt-dlp.exe와 같은 외부 프로그램(예: 명령줄 도구)을 사용할 수도 있지만, Chrome 브라우저 내에서만 해결하고 싶다면 확장 프로그램이나 웹사이트가 더 편리한 방법입니다.
### 1. Chrome 확장 프로그램 사용
Chrome 웹 스토어에는 유튜브 자막 다운로드를 지원하는 다양한 확장 프로그램이 있습니다.
#### 1. 확장 프로그램 설치:
  - Chrome 웹 스토어에서 "YouTube Subtitle Downloader" 또는 이와 유사한 기능을 가진 확장 프로그램을 검색하여 설치합니다.
  - `예시 확장 프로그램`: YouTube Subtitle Downloader 또는 유튜브 자막 추출 등의 이름으로 검색 가능합니다.
#### 2. 자막 다운로드:
  - 자막을 다운로드하고 싶은 YouTube 동영상을 엽니다.
  - 확장 프로그램이 활성화되면, 보통 동영상 플레이어 근처나 브라우저 툴바에 다운로드 버튼이나 옵션이 추가됩니다.
  - 해당 버튼을 클릭하고 원하는 언어와 SRT, TXT, VTT 등의 자막 형식을 선택하여 다운로드합니다.
### 2. 온라인 자막 다운로더 웹사이트 사용
별도의 확장 프로그램 설치 없이 동영상 URL을 붙여넣어 자막을 다운로드할 수 있는 웹사이트도 많습니다.
#### 1. `URL 복사`:
  - 다운로드하고 싶은 YouTube 동영상의 URL을 복사합니다.
#### 2. `웹사이트 접속 및 붙여넣기`:
  - DownSub 또는 유사한 온라인 자막 다운로더 웹사이트에 접속합니다.
  - 복사한 동영상 URL을 입력창에 붙여넣습니다.
#### 3. `다운로드`:
  - 자막을 추출한 후, 원하는 언어와 형식을 선택하여 다운로드합니다.
### 3. YouTube 자체 기능 활용 (텍스트 복사)
파일 형태(SRT 등)가 아닌 순수한 텍스트 형태로 자막을 복사하는 것은 YouTube 자체 기능으로도 가능합니다.
#### 1. `자막/스크립트 열기`:
  - YouTube 동영상 아래의 더보기(...) 버튼을 클릭하고 스크립트 표시를 선택합니다.
  - 동영상 옆에 시간 코드가 포함된 스크립트 창이 나타납니다.
#### 2. 텍스트 복사:
  - 스크립트 창 상단의 **더보기(...)**를 클릭하여 시간 스탬프 전환 옵션을 사용하여 시간 코드를 제거할 수 있습니다.
  - 원하는 텍스트를 드래그하여 복사한 후, 텍스트 편집기(메모장 등)에 붙여넣어 저장합니다.

### Python 스크립트 (번호 붙이기 및 한 줄로 정렬)
이 스크립트는 파일 전체를 하나의 문자열로 읽어 들인 후, 마침표를 기준으로 문장을 분리하고, 불필요한 공백과 줄바꿈을 제거하여 문장을 정리합니다.
```Python
import re
import os

def process_and_number_sentences(filename="readme.txt", start_number=100):
    """
    텍스트 파일에서 마침표(.)를 기준으로 문장을 분리하고,
    100부터 시작하는 일련번호를 붙여서 한 줄로 출력합니다.

    :param filename: 처리할 텍스트 파일의 이름
    :param start_number: 시작할 일련번호
    """
    if not os.path.exists(filename):
        print(f"오류: 파일 '{filename}'을(를) 찾을 수 없습니다.")
        return

    # 1. 파일 내용 전체를 읽어오기
    with open(filename, 'r', encoding='utf-8') as f:
        content = f.read()

    # 2. 불필요한 줄바꿈과 여러 개의 공백을 하나의 공백으로 대체하여 텍스트 정제
    # \n, \r, \t 등의 공백 문자를 일반 공백으로 통합
    clean_content = re.sub(r'\s+', ' ', content).strip()

    # 3. 마침표(.)를 기준으로 텍스트를 문장으로 분리
    # 마침표 뒤에 공백이 있는 패턴을 찾아 분리하고, 마침표는 다시 포함
    # re.split은 마침표를 분리자로 사용하되, 마침표는 다시 결과에 포함하기 어렵기 때문에
    # lookbehind/lookahead 조합을 사용하거나, 여기서는 마침표를 포함하여 분리한 후 정리하는 방식을 사용합니다.
    # 간단하게, 마침표(.)와 그 뒤의 공백을 기준으로 분리합니다.
    sentences = re.split(r'(?<=\.)\s*', clean_content)

    # 마지막 문장이 비어있을 경우 제거 (예: 파일이 .으로 끝났을 경우)
    if not sentences[-1]:
        sentences.pop()

    # 4. 문장마다 일련번호를 붙여서 출력
    print("--- 처리 결과 ---")
    current_number = start_number
    for sentence in sentences:
        # 문장 앞뒤의 공백을 다시 한번 제거
        cleaned_sentence = sentence.strip()

        # 빈 문자열은 건너뜁니다.
        if not cleaned_sentence:
            continue

        # 마침표가 없다면 다시 붙여줍니다 (분리 과정에서 마침표가 제거된 경우 대비)
        if not cleaned_sentence.endswith('.'):
            # 마침표가 없는 경우, 일반적으로 문장의 끝이 아니므로 포함하지 않는 것이 좋지만,
            # 요구사항 예시("AMP Code.")에 맞춰 마침표로 끝나는 문장만 처리하는 것이 더 정확합니다.
            # 하지만 원본 텍스트의 구조상 (예: ...getting\nthe hang of it, another one...)
            # 마침표가 아닌 텍스트도 문단으로 처리해야 할 수 있으므로, 일단 모든 문장을 출력합니다.

            # 요구사항 예시에 맞춰 마침표로 끝나는 문장만 번호 붙여 출력하도록 수정합니다.
            # 단, 예제 텍스트가 여러 줄로 되어 있어 마침표가 포함되지 않은 문장도 생길 수 있습니다.
            # 여기서는 분리된 조각을 '문장'으로 간주하고, 마침표를 강제로 붙이지 않고 원본 유지
            print(f"{current_number}. {cleaned_sentence}")
1            current_number += 1
```

### 🛠️# 개선된 Python 스크립트
아래 스크립트는 파일 전체를 한 줄로 만든 후 마침표를 기준으로 분리하고, 문장 앞뒤의 불필요한 공백을 완전히 제거하여 원하시는 결과를 얻을 수 있도록 수정되었습니다.
```Python
import re
import os

def process_and_number_sentences_v2(filename="readme.txt", start_number=100):
    """
    텍스트 파일에서 마침표(.)를 기준으로 문장을 분리하고,
    100부터 시작하는 일련번호를 붙여서 한 줄로 출력합니다.
    """
    if not os.path.exists(filename):
        print(f"오류: 파일 '{filename}'을(를) 찾을 수 없습니다.")
        return

    # 1. 파일 내용 전체를 읽어오기
    try:
        with open(filename, 'r', encoding='utf-8') as f:
            content = f.read()
    except Exception as e:
        print(f"파일을 읽는 중 오류 발생: {e}")
        return

    # 2. 텍스트 정제: 모든 공백 문자(\s)를 하나의 공백으로 통합 (줄바꿈 제거 및 한 줄화)

    clean_content = re.sub(r'\s+', ' ', content).strip()

    # 패턴: 마침표가 아닌 모든 문자 + 마침표

    # (?:[A-Z]|^) : 대문자나 문장 시작 앞에서만 분리

    # 마침표(.)가 분리자로 사용되므로, 마침표를 다시 문장 끝에 붙여야 합니다.
    # re.split()의 결과는 마침표가 제거된 텍스트 조각들입니다.
    sentences_raw = clean_content.split('.')

    print("--- 처리 결과 ---")

        if not cleaned_sentence:

        # 분리 과정에서 마침표가 제거되었으므로, 다시 마침표를 붙여줍니다.
        final_sentence = cleaned_sentence + '.'

        print(f"{current_number}. {final_sentence}")
```

```Python
import re
import os
import sys

def process_and_number_sentences(filename, start_number=100):
    """
    텍스트 파일에서 마침표(.)를 기준으로 문장을 분리하고,
    지정된 번호부터 시작하는 일련번호를 붙여서 한 줄로 출력합니다.

    :param filename: 처리할 텍스트 파일의 이름
    :param start_number: 시작할 일련번호 (기본값: 100)
    """
    if not os.path.exists(filename):
        print(f"오류: 파일 '{filename}'을(를) 찾을 수 없습니다.")
        # 스크립트 종료 시 파일이 없음을 알림
        sys.exit(1)

    # 1. 파일 내용 전체를 읽어오기
    try:
        # 'r' 모드, UTF-8 인코딩으로 파일을 읽습니다.
        with open(filename, 'r', encoding='utf-8') as f:
            content = f.read()
    except Exception as e:
        print(f"파일을 읽는 중 오류 발생: {e}")
        sys.exit(1)

    # 2. 텍스트 정제: 모든 공백 문자(\s, 줄바꿈 포함)를 하나의 공백으로 통합하여 한 줄로 만듦

    sentences_raw = clean_content.split('.')

    print("--- 처리 결과 ---")
    current_number = start_number

    for raw_sentence in sentences_raw:
        # 문장 앞뒤의 공백 제거
        cleaned_sentence = raw_sentence.strip()

        # 빈 문자열은 건너뜁니다.
        if not cleaned_sentence:
            continue

        # 분리 과정에서 제거된 마침표를 다시 붙여줍니다.
        final_sentence = cleaned_sentence + '.'

        print(f"{current_number}. {final_sentence}")
        current_number += 1
```

```
    print("--- 처리 결과 ---")
    current_number = start_number

    for raw_sentence in sentences_raw:
        # 문장 앞뒤의 공백 제거
        cleaned_sentence = raw_sentence.strip()

        # 빈 문자열은 건너뜁니다.
        if not cleaned_sentence:
            continue

        # 분리 과정에서 제거된 마침표를 다시 붙여줍니다.
        final_sentence = cleaned_sentence + '.'
        lang_mark="e" #-- 'e'nglish, 'k'orean, 한영대본을 sort 로 합칠때 같은번호로 묶기 위한것임.
        print(f"{current_number}{lang_mark}. {final_sentence}")
        current_number += 1


if __name__ == "__main__":
    # 명령줄 인수 확인: 스크립트 이름(sys.argv[0])과 파일 이름(sys.argv[1])이 필요
    if len(sys.argv) < 2:
        print(f"사용법: python3 {sys.argv[0]} <원본_파일_이름>")
        # 파일 이름을 제공하지 않으면 스크립트 종료
        sys.exit(1)

    # 첫 번째 인수를 파일 이름으로 사용
    input_filename = sys.argv[1]

    # 기본 시작 번호 100으로 함수 실행
    process_and_number_sentences(input_filename, start_number=100)
```
