### 1. 사업용/세무 신고가 목적이라면 (강력 추천)
장부 작성뿐만 아니라 나중에 부가세나 종합소득세 신고까지 고려하신다면 다음 앱들이 유리합니다.
* **ERS (간편 영수증 스캐너):** 영수증을 찍으면 날짜, 금액, 업체명은 물론 **부가세까지 자동으로 추출**해줍니다. 엑셀로 내보내기가 가능해 세무사에게 전달하거나 직접 장부를 관리하기에 최적입니다.
* **키퍼 (Keeper):** 개인 사업자들 사이에서 인기가 많습니다. 영수증 촬영은 물론 카드 내역과 연동되어 실시간 손익 분석을 도와주는 전문적인 '간편 장부' 앱입니다.
* **이지샵:** '가계부처럼 쓰는 장부'를 표방하며, 영수증 사진을 올리면 자동으로 장부에 기록해주는 기능을 제공합니다.
### 2. 개인적인 지출 관리(가계부)가 목적이라면
* **편한 가계부:** 국내에서 가장 사용자가 많은 앱 중 하나로, 영수증 스캔 기능이 포함되어 있습니다. 문자(SMS)나 앱 결제 알림과도 연동되어 관리가 매우 편합니다.
* **레픽 (Recpic):** "쓰지 말고 찍자"라는 슬로건처럼 영수증 촬영에 특화된 가계부 앱입니다. 항목별로 자동 분류해주는 기능이 뛰어납니다.
### 3. 글로벌/심플한 관리용
* **ReceiptBox:** 전 세계적으로 사용되는 앱으로 한국어도 지원합니다. 복잡한 세무 기능보다는 영수증을 디지털화하고 카테고리별로 모아보는 기능에 충실합니다.

### 2. 핵심 로직: 영수증 인식 및 저장
아래는 카메라로 영수증을 찍고, 텍스트를 추출한 뒤 "총액"을 찾아 DB에 넣는 핵심 예시입니다.
```dart
import 'package:google_mlkit_text_recognition/google_mlkit_text_recognition.dart';
import 'package:image_picker/image_picker.dart';
import 'database_helper.dart'; // 별도로 만든 DB 헬퍼 클래스

class ReceiptProcessor {
  final TextRecognizer _textRecognizer = TextRecognizer(script: TextRecognitionScript.korean);

  Future<void> scanAndSaveReceipt() async {
    // 1. 이미지 선택 (카메라 촬영)
    final XFile? image = await ImagePicker().pickImage(source: ImageSource.camera);
    if (image == null) return;

    final inputImage = InputImage.fromFilePath(image.path);

    // 2. OCR 실행 (글자 읽기)
    final RecognizedText recognizedText = await _textRecognizer.processImage(inputImage);

    // 3. 데이터 파싱 (간단한 예시: '합계' 또는 '₩' 뒤의 숫자 추출)
    String rawText = recognizedText.text;
    int? amount = _parseAmount(rawText);
    String title = _parseTitle(rawText); // 첫 번째 줄이나 특정 키워드로 상호명 추출

    // 4. SQLite 저장
    if (amount != null) {
      await DatabaseHelper.instance.insertTransaction({
        'title': title,
        'amount': amount,
        'date': DateTime.now().toString(),
      });
      print("저장 완료: $title - $amount원");
    }
  }

  // 간단한 금액 추출 알고리즘 (정규표현식 활용)
  int? _parseAmount(String text) {
    // '합계', '금액', '₩' 뒤의 숫자를 찾는 정규식 예시
    final regex = RegExp(r'(?:합계|금액|TOTAL)\s*[:]?\s*([0-9,]+)');
    final match = regex.firstMatch(text.replaceAll(',', '')); // 콤마 제거 후 검색
    if (match != null) {
      return int.tryParse(match.group(1) ?? '');
    }
    return null;
  }

  String _parseTitle(String text) {
    // 첫 번째 줄을 상호명으로 가정하거나 특정 키워드 앞부분 추출
    return text.split('\n').first;
  }
}
```

### 3. SQLite DB 헬퍼 클래스 (`database_helper.dart`)
데이터를 영구적으로 저장할 간단한 DB 구조입니다.
```dart
import 'package:sqflite/sqflite.dart';
import 'package:path/path.dart';

class DatabaseHelper {
  static final DatabaseHelper instance = DatabaseHelper._init();
  static Database? _database;
  DatabaseHelper._init();

  Future<Database> get database async {
    if (_database != null) return _database!;
    _database = await _initDB('ledger.db');
    return _database!;
  }

  Future<Database> _initDB(String filePath) async {
    final dbPath = await getDatabasesPath();
    final path = join(dbPath, filePath);

    return await openDatabase(path, version: 1, onCreate: _createDB);
  }

  Future _createDB(Database db, int version) async {
    await db.execute('''
      CREATE TABLE transactions (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT NOT NULL,
        amount INTEGER NOT NULL,
        date TEXT NOT NULL
      )
    ''');
  }

  Future<int> insertTransaction(Map<String, dynamic> row) async {
    final db = await instance.database;
    return await db.insert('transactions', row);
  }
}
```

## 01, 개인이 유튜버가 돼서 자기 영상을 올릴때
아마존이나 알리익스프레스에서 판매하는 상품을 리뷰해주는 동영상을 포함시키면, 소정의 금전을 사례로 준다던데 사실인가?

🔋  🔋
### 01' 네, 질문하신 내용은 사실입니다.
개인이 유튜브에 상품 리뷰 영상을 올리고 수익을 얻는 구조는 크게 **'어필리에이트(제휴 마케팅)'**와 **'브랜디드 콘텐츠(협찬)'** 두 가지 방식으로 운영됩니다.

단순히 영상을 올린다고 해서 누군가 바로 현금을 보내주는 것이 아니라, 특정 시스템을 통해 **내가 소개한 상품이 실제로 판매되었을 때** 그 대가로 수수료를 받는 방식이 가장 일반적입니다.

---
### 1. 주요 수익 창출 방식
가장 대표적인 것은 **어필리에이트(Affiliate)** 프로그램입니다.

* **아마존 어소시에이트(Amazon Associates):** 아마존에서 운영하는 제휴 프로그램입니다. 유튜버가 영상 하단 '더보기'란에 상품 링크를 남기고, 시청자가 그 링크를 클릭해 **24시간 이내**에 물건을 구매하면 판매 금액의 일정 비율(보통 **1%~10%**, 카테고리별로 상이)을 수수료로 받습니다.
* **알리익스프레스 어필리에이트(AliExpress Affiliate):** 아마존과 유사하며, 특히 가성비 제품이나 특이한 IT 기기 리뷰어들이 많이 활용합니다. 알리는 아마존보다 수수료율이 높은 경우도 많아 신규 유튜버들이 선호하기도 합니다.
* **쿠팡 파트너스:** 국내에서는 아마존 대신 쿠팡 파트너스를 가장 많이 사용합니다.

### 2. 사례금(현금)을 직접 받는 경우 (협찬)

만약 링크 판매 수수료가 아니라 영상 제작 자체에 대한 **'소정의 금전(광고비)'**을 직접 받는다면, 이는 기업으로부터 직접 의뢰를 받은 **브랜디드 콘텐츠**에 해당합니다.

* 이는 보통 구독자 수가 어느 정도 확보되었거나, 특정 분야(예: 테크, 뷰티)에서 영향력이 있는 유튜버에게 제안이 옵니다.
* 가끔 신제품 홍보를 위해 구독자가 적은 유튜버에게도 제품을 무료로 제공(협찬)하고 리뷰를 부탁하기도 합니다.

### 3. 꼭 알아두어야 할 주의사항 (법적 의무)

수익을 목적으로 리뷰 영상을 올릴 때는 반드시 지켜야 할 법적 규칙이 있습니다.

* **경제적 대가 공개:** 금전적 사례를 받거나 판매 수수료를 받는 링크를 포함했다면, 반드시 영상 내에 **"유료 광고 포함"** 표시를 하거나 설명란에 **"이 포스팅은 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다"**와 같은 문구를 명확히 적어야 합니다.
* **뒷광고 금지:** 대가를 받았음에도 이를 숨기고 '내돈내산(내 돈 주고 내가 산 것)'인 척 리뷰를 하면 공정거래위원회로부터 제재를 받을 수 있으며, 채널의 신뢰도가 크게 떨어집니다.

---

### 요약: 시작하는 방법

1. **가입:** 아마존 어소시에이트나 알리익스프레스 어필리에이트 사이트에서 계정을 만듭니다.
2. **링크 생성:** 리뷰할 상품의 고유 제휴 링크를 발급받습니다.
3. **업로드:** 영상을 올릴 때 설명란이나 고정 댓글에 해당 링크를 넣습니다.
4. **정산:** 누적된 수수료가 일정 금액 이상이 되면 등록한 계좌로 정산받습니다.

라즈베리 파이(Raspberry Pi)나 오렌지 파이(Orange Pi) 같은 싱글 보드 컴퓨터(SBC)를 활용해 자신만의 서버를 구축하는 소위 **'홈랩(Homelab)'**이나 **'셀프 호스팅(Self-hosting)'** 분야는 유튜브에서 꾸준히 사랑받는 테마입니다.

구체적으로 왜 수요가 있는지, 그리고 어떤 점을 강조하면 좋을지 분석해 드립니다.

---
### 1. 왜 수요가 있을까요?
* **비용 절감:** 값비싼 클라우드(AWS, Google Cloud) 대신 저렴한 저전력 보드로 나만의 서비스를 24시간 돌리고 싶어 하는 사용자가 많습니다.
* **개인정보 및 클라우드 탈피:** 구글 드라이브나 노션(Notion) 대신 데이터를 직접 소유하고 관리하려는 수요가 늘고 있습니다.
* **오렌지 파이의 부상:** 라즈베리 파이의 가격 상승과 재고 부족으로 인해 가성비 대안인 **오렌지 파이(Orange Pi)** 설정법을 찾는 사람이 급증했습니다.
* **Wiki.js의 인기:** Wiki.js는 디자인이 유려하고 기능이 강력해, 텍스트 위주의 위키(MediaWiki 등)보다 시각적으로 만족도가 높아 리뷰어와 기록 애호가들에게 인기가 많습니다.

## 03, 그래~ㅎㅎ 그러면, 예를들어 github 에 있는 프로젝트 들에서
좀 만만한걸 골라서 설치하고 운영하는 강의같은걸 만들면 좋을것 같은데, 이런 경우에는 저작권같은게 어떻게 되나? 만든이가 내가 아니니 문제가 될수있지 않을까?

🔋  🔋
### 03' 좋은 질문입니다! 결론부터 말씀드리면,
오픈소스 프로젝트(GitHub 프로젝트)를 설치하고 운영하는 강좌를 만드는 것은 **저작권 문제에서 매우 자유로운 편이며, 오히려 장려되는 활동**입니다.

하지만 몇 가지 **'라이선스(License)'** 규칙과 **'에티켓'**만 지키면 완벽합니다. 핵심 내용을 정리해 드릴게요.

---

### 1. 대부분의 GitHub 프로젝트는 "가르쳐도 된다"

GitHub에 공개된 대부분의 프로젝트는 **오픈소스 라이선스**를 따릅니다.

* **MIT, Apache, BSD 라이선스:** 가장 흔한 라이선스로, 소프트웨어를 사용, 복제, 수정하고 이를 이용해 콘텐츠를 만드는 데 거의 제약이 없습니다. 이 코드를 활용해 강의를 만들고 유튜브 수익을 창출하는 것도 법적으로 문제가 없습니다.
* **GPL 라이선스:** 코드를 수정해서 배포할 때 제약이 있지만, 단순히 설치하고 사용하는 법을 가르치는 강좌 제작에는 아무런 걸림돌이 되지 않습니다.

### 2. 저작권 문제를 피하는 '안전한 강좌' 제작법

직접 만든 소프트웨어가 아니더라도 아래 사항만 준수하면 "훌륭한 교육자"로서 인정받을 수 있습니다.

* **출처 명시 (가장 중요):** 영상 설명란에 해당 GitHub 프로젝트의 **공식 저장소(Repo) 주소**와 **제작자 이름**을 반드시 기재하세요.
> "본 영상은 [제작자 이름]의 [프로젝트명] 프로젝트를 활용하여 제작되었습니다. (링크: [https://github.com/](https://github.com/)...)"


* **공식 로고 사용 주의:** 프로그램 내부에 들어있는 로고는 사용해도 되지만, 마치 본인이 그 브랜드의 공식 파트너인 것처럼 섬네일을 꾸미는 것은 주의해야 합니다. (팬 메이드 강좌임을 밝히는 것이 좋습니다.)
* **유료 강의로 전환 시:** 유튜브가 아닌 별도의 유료 강의 플랫폼(Udemy 등)에 올릴 때는 라이선스를 한 번 더 확인해야 합니다. 하지만 대부분의 'MIT' 라이선스 프로젝트는 유료 강의의 소재로 써도 무방합니다.

### 3. 오히려 원작자가 좋아합니다

많은 오픈소스 개발자들은 자신의 프로젝트가 널리 알려지기를 원합니다.

## 웹에서 사진을 다운로드하려는데 안돼서 화면전체를 캡쳐하고,
gimp 에다가 붙여넣기 했거든. 그리고 필요한 부분만 잘라내서 gimp 의 새창에다가 붙여넣기 하고, 이걸 .webp 로 저장하려고 했더니, gimp: webp 저장실패 - WebP error 'bad image dimensions (maximum 16383x16383)' 이라고 나오네. 그런데 웃기는 것은 .jpg 로 저장하니까 27MB 짜리 파일로 저장이 되고, 연결 프로그램의 사진으로 띄우면 제대로 나오더라고. 해상도를 좀 낮춰도 되니 이걸 .webp 로 저장하려면 어떻게 하면 되나?

### GIMP에서 해당 오류가 발생하는 이유는 WebP 파일 형식이 지원하는 **최대 해상도 제한(16,383 x 16,383 픽셀)**을 초과했기 때문입니다.
JPG는 이보다 훨씬 큰 해상도도 지원하기 때문에 저장이 된 것이지만, 27MB라는 파일 용량을 보면 현재 이미지의 가로 또는 세로 길이가 비정상적으로 길거나 클 가능성이 높습니다. 화면 전체를 캡처한 뒤 필요한 부분만 잘라냈다고 하셨는데, 그 과정에서 캔버스 크기가 조절되지 않았거나 실수로 아주 넓은 영역이 복사되었을 수 있습니다.
해상도를 낮춰서 WebP로 저장하는 가장 확실한 방법들을 정리해 드립니다.

---
### 1. 이미지 크기(해상도) 조절하기
이미지의 실제 픽셀 수를 WebP 제한 범위 안으로 줄이는 방법입니다.
1. GIMP 상단 메뉴에서 **이미지(Image) > 이미지 크기 조정(Scale Image)**을 클릭합니다.
2. **너비(Width)** 또는 **높이(Height)** 중 큰 쪽의 숫자를 확인해 보세요. 아마 16,383보다 큰 숫자가 적혀 있을 것입니다.
3. 수치를 **16,000 이하**로 입력합니다. (보통 웹용이라면 2,000~4,000 픽셀 정도면 충분히 고화질입니다.)
4. 오른쪽의 **체인 아이콘**이 연결되어 있는지 확인하여 비율이 깨지지 않게 합니다.
5. **크기 조정(Scale)** 버튼을 누른 뒤 다시 **내보내기(Export As)**를 통해 .webp로 저장해 보세요.
### 2. 불필요한 여백 잘라내기 (Crop)
혹시 이미지는 작은데 '캔버스'만 크게 잡혀 있는 경우라면 이 방법이 효과적입니다.
1. 도구 모음에서 **자르기 도구(Crop Tool, 단축키 Shift+C)**를 선택합니다.
2. 실제로 저장하고 싶은 영역만 드래그하여 선택합니다.
3. **Enter** 키를 눌러 선택 영역 외의 불필요한 빈 공간을 완전히 제거합니다.
4. 그다음 다시 저장(내보내기)을 시도합니다.
### 3. 레이어를 내용에 맞게 조절하기
붙여넣기를 할 때 레이어가 캔버스보다 훨씬 크게 생성되었을 수 있습니다.
1. 메뉴에서 **레이어(Layer) > 레이어를 내용 크기에 맞추기(Layer to Selection)** 또는 **이미지(Image) > 내용에 맞게 캔버스 크기 조정(Fit Canvas to Layers)**을 실행해 보세요.
2. 현재 실제 데이터가 있는 부분으로 규격이 딱 맞춰집니다.

* **JPG:** 최대 약  픽셀까지 지원하므로 아주 큰 이미지도 저장이 가능합니다. 하지만 27MB라면 웹에서 쓰기에는 너무 무거운 용량이니 위 방법대로 해상도를 줄이시는 것이 좋습니다.
